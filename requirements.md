# FitCafe Website Requirements

## Project Overview
- **Product:** FitCafe Weight Loss Coffee (affiliate product)
- **Domain:** fitcafe.coffee
- **Purpose:** Sales page to drive affiliate conversions
- **Hosting:** GitHub Pages + Vercel compatible
- **Integration:** Affiliate link placeholder (will be provided later)

## Technical Requirements
- **Type:** Static website (HTML/CSS/JS only - no backend needed)
- **Framework:** Vanilla HTML/CSS or lightweight framework (Tailwind CSS recommended)
- **Responsive:** Mobile-first design, fully responsive on all devices
- **Performance:** Fast loading, optimized images, minified assets
- **SEO:** Basic SEO optimization (meta tags, semantic HTML, Open Graph)
- **Browser Support:** Modern browsers (Chrome, Firefox, Safari, Edge)

## Content Structure
Use the sales copy from `../TASK-20260321-003-GEM-SalesPage.md` as the base content. Structure should follow the HTML sections provided by Gem:

1. Headline
2. Subheadline  
3. Opening Story/Relatability
4. Product Introduction
5. How It Works (Ingredients)
6. Benefits Section
7. Social Proof/Testimonials
8. Guarantee Section
9. CTA Section

## Design Guidelines
- **Tone:** Warm, trustworthy, conversational (not salesy or aggressive)
- **Colors:** Professional, clean aesthetic - consider coffee-inspired colors (browns, creams) with accent colors for CTAs
- **Typography:** Clean, readable fonts (Google Fonts recommended)
- **Imagery:** Placeholder for product images (will be added later)
- **CTA Buttons:** Prominent but not overwhelming, use soft language like "Try FitCafe" instead of "Buy Now"

## Integration Points
- **Affiliate Link:** Placeholders for affiliate links in CTA buttons and any relevant links
- **Analytics:** Basic Google Analytics placeholder (optional)
- **Social Sharing:** Open Graph meta tags for social sharing

## Deliverables
- Complete HTML file (`index.html`)
- CSS file(s) (`style.css` or Tailwind setup)
- JS file(s) if needed (`script.js`)
- Assets folder structure for images/fonts
- README.md with deployment instructions for GitHub Pages and Vercel

## Notes
- This is an affiliate site - no payment processing needed
- Focus on trust-building and conversion optimization
- Keep code clean and well-commented for future maintenance
